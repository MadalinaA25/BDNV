ACID vs BASE

ACID (PostgreSQL)

  • Atomicity - All or nothing
  • Consistency - Valid states only
  • Isolation - No interference
  • Durability - Survives crashes

Ideal pentru: Orders, Payments, Inventory

BASE (MongoDB - optional)

  • Basically Available
  • Soft State
  • Eventual Consistency

Ideal pentru: Sessions, Logs, Analytics

Concluzii Finale

  • Nu exista "best database" - depinde de use case
  • PostgreSQL: ACID obligatoriu pentru tranzactii
  • MongoDB: Flexibilitate si scalare orizontala
  • CAP Theorem: Ambele sunt CP by default
  • Deployment-ul afecteaza major performanta
  • Polyglot Persistence = best practice modern
